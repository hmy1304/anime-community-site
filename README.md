# anime-community-site

이 사이트는 제가 이번 수업에서 배운 것을 그대로 활용하여 만든 실험적인 애니메이션 커뮤니티 웹사이트입니다.

## 프로젝트 개요

이 웹사이트는 완성도보다는 제가 수업시간에 배우면서 중간에 '이렇게 사용해보면 어떨까?'라는 생각이 나는 것을 그대로

활용하여 만들었으며, 미디어쿼리나 modal을 활용한 댓글작성창을 구현, 그리고 페이지에서 페이지, 사이트에서 사이트로 가는 링크기능에

중점을 두어 만들었습니다.

## 화면 구현

* 로그인 페이지는 헤더의 디자인은 약간 다르지만 모든 페이지를 상단의 헤더와 밑에 메인 영역으로 나누어서 제작하였습니다.

* index페이지는 메인 안에 요일div와 anime-list div로 위아래로 나누었으며, anime-item 마우스를 올렸을 때 변화를 주도록

  hover를 주었습니다.

* board는 총 여섯개로 만들었으며, 각각 페이지 마다 내용이 다르게 들어갔고, 외부 사이트로 바로가기, 댓글 작성, 좋아요 누르기, 그리고

  댓글보기를 구현했으며, 수업 때 배운 modal을 만드는 방식을 참고하여 댓글 작성 기능을 만드는 것에 중점을 두었습니다. 

## HTML 작성 시 쓴 점

* 클래스를 하나하나 지정하여 제3자가 봐도 이해하기 쉽도록 딱 봐도 이건 뭐다라는 것을 알 수 있는 이름으로 짓는 것을 중요하게 봤습니다.

## CSS 작성 시 신경 쓴 점

* 이미지들의 크기나 화면 사이즈에 따라서 변화는 사이즈 조정에 중점을 두어 작성하였습니다.

* 버튼이라던가 클릭할 수 있는 요소에 hover요소를 추가하여 최대한 지금 제가 최대한 할 수 있는 작동시킬 수 있는 것들을 구현하였습니다.

## 구현 중 문제와 해결

## 실습을 통해 확인한 점

* 간단한 페이지 구형에도 생각보다 많은 양의 코드가 들어간다는 것을 새삼 다시 깨닳았습니다.

* 생각보다 신경써서 구현해야 하는 것들이 많아 제가 너무 쉽게 생각하고 있었다고 반성했습니다.

* 중간중간에 java-script를 넣어서 넣고 싶은 기능을 추가하였는데 java-script가 있고 없고의 차이가 크다는 것을 알게 되었습니다.

## 개선할 점

* 이번 실습을 하면서 제가 미흡하게 구현한 것이라고 하면 index페이지에 요일별로 애니메이션을 정렬하는 기능을 추가하지 않은 것과

  사이트의 전체적인 디자인이 많이 미흡한 것 같습니다.

* 추가적으로는 애니메이션 기능을 좀 더 활용하는 게 어땠을까라는 생각이 듭니다.
